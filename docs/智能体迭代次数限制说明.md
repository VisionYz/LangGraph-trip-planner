# 智能体迭代次数限制说明

## 问题背景

LangGraph 中的智能体（Agent）在执行任务时，会循环调用 LLM 和工具：

```
LLM 决策 → 调用工具 → 获取结果 → LLM 再决策 → 再调用工具 → ...
```

如果没有设置迭代上限，智能体可能陷入**无限循环**（比如工具返回结果不符合预期，LLM 反复重试），导致：
- **Token 消耗失控**：每轮循环会携带之前所有消息历史，token 消耗指数级增长
- **响应时间过长**：用户等待几分钟甚至更久
- **费用暴涨**：一次请求可能消耗数十万 token

## 两层限制机制

### 1. 智能体级别：`AGENT_MAX_ITERATIONS`

控制每个子智能体（景点搜索、天气查询、酒店搜索、行程规划）调用工具的最大轮数。

- **配置位置**：`backend/.env`
- **默认值**：`3`
- **生效方式**：通过 `recursion_limit` 参数传入 `agent.invoke()`

```python
# trip_planner_graph.py 中的调用方式
result = self.attraction_agent.invoke(
    self._prepare_agent_input(query, []),
    config={"recursion_limit": settings.agent_max_iterations * 2 + 1}
)
```

> **为什么是 `* 2 + 1`？**
> LangGraph 的 ReAct 智能体中，每次"工具调用"实际包含 2 个步骤：
> 1. LLM 决策（选择调用哪个工具）
> 2. 工具执行（实际调用工具并返回结果）
>
> 所以 3 次迭代 = 6 个步骤 + 1 个初始步骤 = `recursion_limit=7`

### 2. 工作流图级别：`GRAPH_RECURSION_LIMIT`

控制整个旅行规划工作流（包含所有节点）的最大执行步数。

- **配置位置**：`backend/.env`
- **默认值**：`25`
- **生效方式**：通过 `config` 参数传入 `graph.invoke()`

```python
# trip_planner_graph.py 中的调用方式
final_state = self.graph.invoke(
    initial_state,
    config={"recursion_limit": settings.graph_recursion_limit}
)
```

## 配置方法

在 `backend/.env` 中设置：

```env
# 每个智能体调用工具的最大轮数（建议 2-5）
AGENT_MAX_ITERATIONS=3

# 整个工作流图的最大节点执行次数（建议 15-30）
GRAPH_RECURSION_LIMIT=25
```

在 `backend/app/config.py` 中定义：

```python
class Settings(BaseSettings):
    # 智能体配置
    agent_max_iterations: int = 3
    agent_temperature: float = 0.7
    agent_timeout: float = 30.0

    # LangGraph 图配置
    graph_recursion_limit: int = 50
```

## Token 消耗估算

| 场景 | 每个智能体迭代次数 | 4个智能体总消耗 |
|------|-------------------|----------------|
| 无限制（危险！） | 20+ 轮 | 400K+ tokens |
| `AGENT_MAX_ITERATIONS=3` | 最多 3 轮 | ~36K tokens（最坏） |
| 正常执行 | 1-2 轮 | ~10K-20K tokens |

## 涉及的文件

| 文件 | 改动说明 |
|------|---------|
| `backend/.env` | 新增 `AGENT_MAX_ITERATIONS` 和 `GRAPH_RECURSION_LIMIT` |
| `backend/app/config.py` | 新增 `graph_recursion_limit` 配置项 |
| `backend/app/workflows/trip_planner_graph.py` | 所有 `agent.invoke()` 和 `graph.invoke()` 传入 `recursion_limit` |
| `backend/app/agents/langgraph_agents.py` | 导入 `settings` 配置 |
| `backend/app/services/llm_service.py` | 从 config 读取 `temperature`/`timeout`，`max_retries` 降为 2 |

## 超出限制时的行为

当智能体达到迭代上限时，LangGraph 会抛出 `GraphRecursionError` 异常，该异常会被各节点的 `try/except` 捕获，触发错误处理流程，最终返回一个**备用计划**（fallback plan）给用户，而不是无限等待。
